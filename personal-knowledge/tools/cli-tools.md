# Shell

I use **Bash** — it's ubiquitous across every platform I touch. No customizations worth a separate config; the portability of vanilla Bash is the feature.

# Standard CLI Tools

| Tool | Purpose |
|------|---------|
| `tmux` | Terminal multiplexer — persistent sessions, split panes, detach/reattach |
| `vim` | Terminal-based text editor for quick edits |
| `git` | Version control |
| `curl` | HTTP requests, API debugging, file downloads |
| `cat` | File output, concatenation |
| `find` | Filesystem search |
| `grep` | Pattern matching and text search |
| `jq` | JSON processing and transformation |
| `for` | Shell loops for batch operations |

# AI Coding Agents

| Tool | Role |
|------|------|
| **Codex** (OpenAI) | AI coding agent — used within Devboxer cloud sessions and occasionally directly |
| **Claude** (Anthropic) | AI coding agent via Claude Code CLI |
| **Droid CLI** (Factory) | AI coding agent — used alongside the other agents for different strengths |
| **Forge** | AI coding agent CLI |
| **Pi CLI** (Earendil) | AI coding agent — my primary local agent for project work, extensions, and this OKF bundle |

# Usage Pattern

I don't use a single AI agent for everything. Different agents have different strengths, model backends, and tool ecosystems. I select based on the task — Claude for deep reasoning, Pi for extension-heavy project work, Codex for Devboxer cloud sessions, Droid and Forge as alternatives.
